Tech Stack Approaches

How to build the custom Crux puzzle — all options deploy easily to GitHub Pages

A

Single HTML File (Recommended)

One self-contained index.html with vanilla HTML, CSS, and JavaScript. No build step, no dependencies, no framework. Just open the file or push to GitHub Pages.

Pros

  • Zero setup — works immediately
  • Can also send as a file attachment
  • No build tools, no node_modules
  • Easy to debug and iterate

Cons

  • All code in one file (but it's a small project)
  • No hot-reload dev server (just refresh browser)
B

Vite + TypeScript

Lightweight dev setup with Vite for hot-reload and TypeScript for type safety. Separate files for grid logic, rendering, and interaction. Builds to static files for deployment.

Pros

  • Hot-reload during development
  • Type safety catches bugs early
  • Clean file separation

Cons

  • Requires Node.js + npm install
  • Build step needed for deployment
  • More setup for a one-off project
C

React / Vue SPA

Full component framework with state management. Overkill for a single hardcoded puzzle, but familiar if you use these at work.

Pros

  • Component-based architecture
  • Rich ecosystem

Cons

  • Heavy for a single-page puzzle
  • Complex build pipeline
  • Unnecessary abstraction
